## 창의1 · 실시간 로그 이상탐지

실습1은 로그 20만 줄을 다 읽은 뒤에 집계하는 방식입니다. 그런데 실제 서비스라면
다 읽고 나서가 아니라 흘러가는 중에 장애를 알아채야 한다고 생각해서 만들었습니다.

- `deque`로 슬라이딩 윈도우를 만들어 최근 300건의 5xx 비율을 계속 감시
- 평소 8%인데 12%를 넘으면 경보 + 그 순간 오류가 몰린 경로 스냅샷
- 같은 장애로 경보가 쏟아지지 않게 쿨다운 적용
- 끝나면 시간대별 트래픽을 터미널에 막대로 출력

실행: `python Advanced/창의1_실시간_로그_이상탐지.py`

## 창의2 · SKALA 매점 재고관리

"내가 만약 SKALA 매점 사장이라면?"에서 시작했습니다. 커피, 얼음컵, 과자 같은
상품을 놓고 재고 부족·품절·유통기한 임박·가격 오류를 자동으로 찾아내고,
관리자에게 발주·할인·데이터 수정 같은 할 일을 알려줍니다.

- 재고와 판매량을 한눈에 보는 HTML 대시보드
- 관리자 경고 목록 CSV + 처리 완료 체크
- pytest 4개로 검증

실행: `python Advanced/창의2_매점_재고관리.py` · 테스트: `cd Advanced && pytest -v`

## 창의3 · 이탈 조기경보

모델을 만드는 것과 쓰는 것은 다른 일이라 생각해서, 종합실습2에서 저장한
`churn_model.joblib`을 실제로 불러와 이번 달 고객 200명을 점수화했습니다.

핵심은 **확률 순이 아니라 기대손실(이탈확률 × 월요금) 순**으로 줄 세운 것입니다.
확률 0.9인 저가 고객보다 확률 0.5인 고가 고객을 먼저 잡는 게 회사엔 이득이니까요.
한정된 상담원을 어디에 투입할지 정하는 관점입니다.

- `joblib.load`로 전처리 포함 Pipeline 통째로 로드 → 새 고객 바로 예측
- 위험도 3단계 분류 + 오늘 전화할 TOP 10
- 상담팀 배포용 CSV (엑셀에서 한글 안 깨지게 `utf-8-sig`)

실행: `python Advanced/창의3_이탈_조기경보.py`
(먼저 `python Total2/종합실습2_EDA_통계_ML.py`로 모델을 만들어야 합니다)

## 창의4 · SKALA 광주 3반 진도 코치

HTML/CSS 주차에 쓰던 진도 체크표를 내보낸 JSON이 남아 있길래, 이번에 배운 걸
우리 반 데이터에 그대로 써봤습니다. 교수님 입장에서 궁금한 건 결국 세 가지라고
생각해서 그 순서로 답을 만들었습니다.

**① 다들 어디서 막혔나** — 과제를 배운 순서대로 세워놓고 남은 인원을 세는 퍼널을
그렸습니다. day2의 JavaScript 기초와 비동기 모듈에서 두 명씩 빠지고, 상품 카드
갤러리부터는 완료율이 20%까지 떨어지는 게 보입니다.

**② 누구를 먼저 도와야 하나** — 적게 한 순으로 줄 세우면 성의가 없다고 생각해서,
미완 비율(50) + day2 미완(20) + 본인 도움요청(30)으로 위험점수를 만들었습니다.
후반부에서 못 따라오는 쪽에 가중치를 더 줬습니다.

**③ 전공/비전공 차이가 진짜인가** — 이 과제에서 제일 많이 고민한 부분입니다.
종합실습2에서 쓴 카이제곱을 그대로 가져오면 **틀립니다**. 우리 반은 10명뿐이라
2×2 표의 기대빈도 최솟값이 0.8까지 떨어져서 카이제곱 근사가 깨지기 때문입니다.
그래서 소표본에 맞는 **Fisher 정확검정**과, 정규성 가정이 필요 없는
**Mann-Whitney U 검정**으로 바꿔서 계산했습니다.
두 검정 모두 p > 0.05 → **전공 여부로 진도 차이를 설명할 수 없다**는 결론입니다.
실제로 진도가 가장 느린 학생도 전공자였습니다.

결과는 `output/skala_progress_dashboard.html` 대시보드로 정리했고,
코칭 명단은 CSV로도 뽑았습니다.

실행: `python Advanced/창의4_SKALA_진도율_분석.py`

> 진도 데이터의 이름은 체크표 앱의 예시 이름(연예인 이름)이라 실제 수강생
> 개인정보는 들어 있지 않습니다.
