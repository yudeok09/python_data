유덕현 자기평가

실습1에서는 20만 건의 웹 로그를 한꺼번에 메모리에 올리지 않고 제너레이터로 한 줄씩 처리했다. 상태 코드, 접속 경로, 시간대, IP를 한 번의 순회로 집계했으며, reduce 방식과 일반 반복문의 결과가 같은지도 확인했다. 또한 tracemalloc로 메모리 사용량을 비교해 제너레이터 방식의 효율성을 직접 확인했다.

실습2에서는 Pydantic을 이용해 중첩된 사용자 데이터를 검증하고, 정상 36건과 오류 4건을 분리해 오류가 발생한 필드와 이유를 확인할 수 있게 작성했다.

실습3에서는 asyncio와 Semaphore를 이용해 60건의 데이터를 동시에 수집하고, 동기 방식과 실행 시간을 비교했다. 일시적인 오류가 발생해도 지수 백오프 방식으로 재시도하고, 타임아웃과 최종 실패 처리 기능을 추가했다. 모의 수집뿐 아니라 httpx를 이용한 실제 HTTP 수집도 구현해 정상적으로 동작하는 것을 확인했다.

종합실습1은 데이터 수집, 검증, 저장 과정을 Extract, Transform, Load로 나누고 CSV와 Parquet 파일로 저장했다. 각 단계가 제대로 동작하는지 pytest로 확인했으며 전체 테스트와 Ruff 코드 검사를 통과했다.

창의 과제로는 ‘내가 만약 SKALA 매점 사장이라면?’이라는 상황을 생각해 커피, 얼음컵, 과자, 음료 등의 상품 관리 프로그램을 만들어 보았다. 재고 부족, 품절, 유통기한 임박, 가격 오류 등의 문제를 자동으로 찾고 관리자에게 발주, 할인, 데이터 수정 같은 해결 방향을 알려주도록 구현했다. 재고와 판매량을 그래프로 확인할 수 있는 대시보드와 처리 완료 체크 기능도 추가해 실제 매점 관리에 활용할 수 있도록 확장했다.
